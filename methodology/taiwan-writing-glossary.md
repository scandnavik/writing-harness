# 台灣寫作 glossary

跨寫作 skill 通用的 hard rules 與用字對照表。**所有寫作 skill 在 SKILL.md 與 harness 檢查腳本中都應引用此檔**。新增規則先加在「驗證期」段落，觀察一段時間無推翻再升正式條目。

> 這份 glossary 是針對**台灣繁體中文**語境調校的。如果你的目標語境是別的（中國大陸簡中、香港繁中、純書面學術中文），把 §2 的對照表整段換成你那邊的判準即可，方法論不變。

## §1 標點與排版

### 1.1 標點全形強制

中文行文一律用全形標點，半形視為違規。

| 全形（正確） | 半形（禁用） | 用途 |
|------------|------------|------|
| `，` | `,` | 逗號 |
| `。` | `.` | 句號 |
| `：` | `:` | 冒號 |
| `；` | `;` | 分號 |
| `？` | `?` | 問號 |
| `！` | `!` | 驚嘆號 |
| `「」` | `""` | 引號（中文用「」，不用 ""） |
| `（）` | `()` | 括號（中文文意用全形） |

**例外**：純英文句、程式碼、檔名、URL、版本號（4.7、v1.2、settings.json）保留半形。

**為什麼**：半形 `,` 在中文行文中看起來怪、不專業，是 AI 預設輸出的明顯指紋之一。

**驗收 regex**：`[一-龥]\s*[,\.\:;\?\!]\s*[一-龥]` 命中即報違規。

### 1.2 破折號「——」禁令（hard rule）

寫作不准用破折號，任何來源派工指令都不能覆蓋。

**怎麼改**：逐處判斷取代：
- **句號**：前後是完整意思可以獨立站的兩句 → 拆兩句
- **逗號**：前後是輕度補充或連接 → 改逗號
- **冒號**：後面是前面的展開、舉例、定義 → 改冒號
- **括號**：後面是補充解釋或插入語 → 用全形括號（）
- **重寫**：上面都不適合 → 重寫句子保留原意

不准用 `…`、英文 hyphen、書名號等湊數。已完稿驗收時，破折號命中直接列違規，不准理由保留。

**為什麼**：破折號是 AI 生成中文最氾濫的連接符，過量使用會讓行文有明顯的「機器節奏」。

**驗收 regex**：`——|--|—` 命中即違規。

## §2 用字遣詞（大陸 → 台灣）

> 以下對照表是台灣語境判準。換語境時整段替換。

### 2.1 詞彙對照

| 大陸用語 | 台灣用語 | 觸發語境 |
|---|---|---|
| 範式轉換 | 典範轉移 | paradigm shift 譯名 |
| 視頻 | 影片 | 多媒體 |
| 在線 | 線上 | 數位狀態 |
| 網絡 | 網路 | internet |
| 批量 | 批次 | 量化處理 |
| 項目 | 專案 | project |
| 數據 | 資料 | data |
| 軟件 | 軟體 | software |
| 信息 | 資訊 | information |
| 質量 | 品質 | quality |
| 文件 | 檔案／文件（看語境） | file vs document |
| 平臺 | 平台 | 字形差異 |
| 公眾號 | 粉專／帳號 | 社群媒體 |
| 視頻號 | 短影音帳號 | — |
| 默認 | 預設 | default |
| 鏈接 | 連結 | link |
| 用戶 | 使用者／用戶（兩者皆 OK） | user |
| 互聯網 | 網路 | internet |

### 2.2 動詞 audit 表（最常踩雷）

寫完後逐動詞掃過，比對下表：

| 偏大陸 | 台灣用語 | 觸發情境 |
|---|---|---|
| 接下（任務／事情） | 接下來／接住 | 「把每件事都接下」→「接下來」 |
| 一條條清掉 | 一條條打掉 | 證據／障礙 |
| 對接 | 接洽／對應／串接 | 客戶接洽、跨部門對應 |
| 推來的案子 | 找上門的案子／接到的案子 | 業務情境 |
| 卡上的是 | 排上的是 | 排程／隊列 |
| 搞反／搞 X | 弄反／弄 X | 「搞」字在大陸用得頻繁 |
| 讀漏了 | 漏看了／漏掉 | 閱讀理解 |
| 沒細答 | 沒講細／沒交代清楚 | 訪談／書評 |
| 裁掉 | 砍掉 | 內容／段落 |
| 躺平 | 擺爛 | 失去動力的形容 |
| 自承 | 自己說／坦言 | 引用本人說法（沒有認罪語境時） |
| 撐（士氣／態度／處境等抽象主詞） | 維持／面對／走過 | 「員工士氣怎麼撐」→「員工的士氣怎麼維持」 |

### 2.3 動詞通則（從上表抽出）

1. **「撐」用在抽象主詞**（士氣、態度、處境）→ 換「維持」「面對」「走過」
2. **「搞」字當動詞前綴** → 換「弄」
3. **「接 X／推 X」單字動詞接事務** → 加助詞變多字（接下來／找上門）
4. **「裁／清」當「砍掉」用** → 直接寫「砍掉」「打掉」
5. **抽象詞前的「的」不要省** → 「員工士氣」加「的」變「員工的士氣」更台灣
6. **長句句中加「，」斷氣口** → 不要兩個分句用「不是 X，是 Y」一氣到底
7. **書面化動詞**（自承、自陳）→ 沒有正式語境就用口語（自己說、坦言）

## §3 術語白話化

### 3.1 API 技術術語禁令（hard rule）

解釋技術概念給一般讀者時，不要用「打 API call」「呼叫 API」這類術語。技術文件可以保留，面向讀者的解釋一律白話。

**範例**：
- ❌ 打 API call → ✅ 在背景偷偷幫你問一個問題
- ❌ 呼叫 API → ✅ 偷偷送請求出去

**為什麼**：一般讀者覺得這種講法超難懂。

### 3.2 英文技術詞中文化（科普文／社群文）

科普文預設中文化。只有翻譯會失真的極少數詞保留原文（例：Architect、Figure 6、free-form）。

| 英文 | 中文 |
|---|---|
| ticket / bug | 任務 |
| tech guy | 科技人 |
| junior | 初階 |
| context（能力背景語境時） | 經驗需求 |
| prompt | 提示詞 |
| code | 程式碼 |
| agent | 看情境，配 Agentic AI 觀念啟蒙時保留 |

**驗收**：grep 正文英文詞密度，>1% 就 warn。

**為什麼**：科普文讀者是初學者，已經被主題墊高一層門檻，英文詞再墊一層會勸退。

## §4 句型與結構

### 4.1 「不是 X，是 Y」否定翻轉句型

**規則**：文案寫作禁止「不是 X，是 Y」「不需要 X」「他不是 X，他是 Y」等先否定再翻轉的 AI 慣性句型。直接講正面事實。

**範例**：
- ❌「不是範本，是根據你的業務做的」→ ✅「根據你的業務，從頭做出來的那一個」
- ❌「他不是工程師，他是⋯」→ ✅「他是⋯」
- ❌「不需要寫程式碼」→ ✅「用對話就能完成」

**密度上限**：同一篇文章用超過 2 次「不是 A，是 B」就要輪替：
- 「不見得 A，他們只是 B」
- 「而是」加轉折
- 「並不代表」＋補說明

**為什麼**：先假定一個事實，再突破直覺認知，最後給答案——這個結構太 AI 了。機械重複會讓文章的結構感壓過內容，像 AI slop 的二元對比模板。

**驗收 regex**：`不是.{1,15}是` 出現次數 > 2 就換句。

### 4.2 因果段補連接詞，不只靠冒號硬接

寫因果邏輯時，不要只用冒號跳到結果。補「因為／才／所以」讓讀者跟上。

**範例**：
- ❌「是有 Architect 這種角色：⋯⋯」
- ✅「但因為有 Architect 這種角色：才讓執行的結果被放大。」

**為什麼**：冒號適合列舉或定義，不適合因果推導。純冒號跳接是邏輯鬆的訊號。

### 4.3 段落標題優先主述句，少用否定懸念

段標題偏好「變快並非紅利」（主述肯定）勝過「紅利不是變快」（否定懸念）。

**為什麼**：否定句型有被動感、像社群鉤子放在正文段標出戲。主述句更有主張感，讀者一眼看到結論。

## §5 動詞血肉與收尾

### 5.1 收尾保留動詞，不為 punch line 抽掉具體動作

具體動詞（採坑／優化／刪掉重來）比抽象名詞（使用經驗）更能落地。

**規則**：收尾前檢查，原始素材裡的具體動詞要保留至少 60%。把 3 個動詞濃縮成 1 個抽象名詞，視為損失。

**為什麼**：把「採坑、優化、刪掉重來」抽象成「使用經驗」會丟掉三個動詞。抽象金句讀者無感，動詞能看見畫面。

## §6 驗收 hook regex 建議

寫作 skill 完稿後跑下列檢查（`scripts/taiwan-style-check.py` 已實作大部分）：

| 檢查項 | regex／方法 | 違規處理 |
|---|---|---|
| 半形標點 | `[一-龥]\s*[,\.\:;\?\!]\s*[一-龥]` | 全部換全形 |
| 破折號 | `——\|--\|—` | 取代或重寫 |
| 「不是 X，是 Y」密度 | `不是.{1,15}是` count > 2 | 換句型 |
| 英文詞密度（科普文） | 半形英文詞／總字數 > 1% | 中文化 |
| 動詞 audit | 比對 §2.2 表逐詞掃 | 換台用動詞 |
| API 術語 | grep `API call\|呼叫 API\|API 呼叫` | 換白話 |
| 大陸用語 | 比對 §2.1 表 grep | 換台灣用語 |

各 skill 在 harness 階段把上述檢查包成 Python／JS function，命中報告位置 + 建議改法。
