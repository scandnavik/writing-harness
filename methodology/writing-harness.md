# 寫作 Harness（中文對外／知識資產長文）

> 操作型 SSoT。所有面向人眼閱讀的中文長文（科普／社群貼文／案例／報告書敘述段／提案故事段／客戶訊息／教學攻略）**必須**穿過下面三站，未過三站不得宣告完成。
> 不適用：表格、yaml、程式碼、清單頭尾、純機械骨架（標 `draft-skeleton`）。

這是一套「品質閘」(quality gate) 的設計：把「會不會寫成 AI slop」這件事，從事後憑感覺修，前移成動筆前、草稿後兩個固定檢查點。最便宜的返工時機是「還沒寫」和「剛寫第一段」，三站就是卡在那兩個點上。

---

## S0 輸入閘（動筆前，最高槓桿）

寫之前先問：這篇要不要**真實場景／案例／引述對白／數據**？

- 要 → 有沒有可用的真實（可匿名）素材？
  - 有 → 進 S1 區的撰寫
  - **沒有 → STOP。跟需求方要最小素材（對方真的說／問了什麼、出什麼錯、後果）。禁止自行編造再標「示意／假設／placeholder」蒙混。**
- 不要（純框架／觀點文）→ 仍禁止虛構案例當佐證；無真例就不寫那個例子。

被打回「這是編的」時：不要再生一個更好的編造，停止產出，把缺口當輸入需求丟回需求方。

> **為什麼這站最重要**：AI slop 最致命的不是用詞，是「煞有介事地編一個看似具體的例子」。一旦容許編造，後面兩站再嚴也只是把假內容磨得更光滑。S0 是唯一能擋掉這個的點。

---

## S1 機械閘（草稿後，純腳本 0 LLM）

```
python ~/.claude/skills/writing-harness/scripts/taiwan-style-check.py <file.md>
```

exit 0 才算過。涵蓋：破折號／半形標點／陸用詞／API 術語／開場推銷詞／「不是 X 是 Y」密度 ≤2／**結論引導‧雜訊框架詞**（理由很 X、問題出在、真正難／值錢、這正是重點、值得注意的是、又 X 又 Y、句首「其實／老實說／我記得／說真的」等）。

命中即修，修完重跑，不得跳過。Edit 後重讀該行驗證。

> 機械閘只抓「正則抓得到」的硬規則，且刻意收高精度子集（低誤報），模糊的留給 S2 人判。腳本本體與規則一一對應，新增可正則化的規則就往腳本補 check function。

---

## S2 判斷閘（草稿後，強制自審＋留證）

腳本抓不到的（結構、聲音、敘事姿態、台灣語感、編造）只能人判。**逐項自答並把結果貼進回覆**，才能宣告完成。

### 2a 結構／聲音 5 問
1. 開場有具體場景／時間／人，還是全景泛論？
2. 有一個「我自己的」、會被反駁的主張當主軸？
3. 段落詳略刻意不平均（重點停留、其餘掃過），還是 N 段等長罐頭？
4. 收尾是具體斷言，還是空心金句（刪掉看內容掉不掉血）？
5. 全篇有至少一個只有當事人講得出的真實（可匿名）例子／數字？

任一答不出 → 還是 AI 文，**重寫骨架不是潤稿**。

### 2b 敘事姿態
- 評價詞前置自查：每個「很／最／真正／幾乎／太＋評價詞」「理由很 X」「問題出在 Y」，問「刪掉它，讀者靠前文推得出來嗎？」推得出 → 刪（讀者自己得出＝敘事成功）；推不出 → 補場景不是補形容詞。
- 雜訊框架詞清零：句首「其實／老實說／坦白講／說真的／我記得」這類宣告真實感的填充，零承載，刪。角色講話直接「X 說」。
- 收束句界線：可命名「剛剛發生什麼」，不可在它發生前先定性。

### 2c 台灣在地語感
- 有對白／口語處唸出來，辨認得出是台灣人講的才留。中小企業老闆語境（register）錨點範例：問真實會問的問題（「到底可以幫我們在哪邊加分」）＋成本焦慮口語化（「有沒有更省錢的作法」）＋省主詞口語強調（「不然請大會計師事務所超級貴」）。generic 書面普通話＝AI 預設腔，重寫。

### 2d 句子層 9 類人眼語病（默讀掃）
斷句不清／修飾鏈繞口／缺動詞／缺受詞／語序怪／指代不清／詞性誤認／口語縮寫不當／擬人化過度。

## 留證格式（宣告完成時貼這個）

```
S0 輸入閘：真實素材=有／無（無 → 已 STOP 要素材，未產出）
S1 機械閘：taiwan-style-check exit 0 ✅（或列已修項）
S2 判斷閘：
  2a 5 問：①… ②… ③… ④… ⑤…（逐題一句話佐證）
  2b 評價詞／雜訊詞：已掃，列刪改處或「無」
  2c 台灣語感：對白唸過／無對白
  2d 9 類語病：默讀過，列修處或「無」
```

三站任一未過或留證缺項＝未完成，不得回報 done。

---

## 為什麼是「Harness」而不是「prompt 裡寫幾條規則」

把規則塞進 prompt 有兩個失效模式：模型會選擇性遵守（純文字描述的規則最容易被跳過），而且事後沒有任何證據能證明它真的做了。Harness 的差別是：

1. **機械閘是程式，不是承諾** — exit code 不會說謊。
2. **判斷閘要留證** — 逐題自答貼回覆，等於強制 self-attestation，比「我檢查過了」可稽核。
3. **可加閘、只升不降** — 每次被打回的新錯誤，先問「正則抓得到嗎」，抓得到就往 S1 腳本加一條，抓不到才寫成 S2 的判斷項。規則庫單向長大，品質地板只升不降。
